Data Engineering Project 3 - RAG Pipeline

Retrieval-Augmented Generation with LangChain, Groq & HuggingFace

Built a fully functional RAG (Retrieval-Augmented Generation) pipeline using LangChain's LCEL (LangChain Expression Language). The pipeline loads documents, splits them into chunks, embeds them with HuggingFace, stores them in a Chroma vector store, and answers questions using the Groq-hosted Llama 3.1 model — all in a clean, composable chain.

LangChain Groq (Llama 3.1) HuggingFace Embeddings Chroma Python
  • Loaded and chunked documents using TextLoader and RecursiveCharacterTextSplitter for efficient retrieval.
  • Generated semantic embeddings with HuggingFace all-MiniLM-L6-v2 and persisted them in a Chroma vector store.
  • Composed a full RAG chain using LCEL — retriever → prompt → LLM → output parser — in a single declarative pipeline.
  • Powered responses with Groq's llama-3.1-8b-instant model for fast, accurate question answering over custom documents.
